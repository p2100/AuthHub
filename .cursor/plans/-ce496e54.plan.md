# 修复权限管理页面按钮无响应问题

## 问题根因

在 `/Users/leo/code/work/AuthHub/frontend/src/pages/Permissions/PermissionList.tsx` 中：

1. **第263-265行**：创建权限按钮缺少 `onClick` 事件
2. **第282-284行**：创建路由规则按钮缺少 `onClick` 事件  
3. 缺少 Modal 组件来处理创建操作
4. 缺少状态管理来控制 Modal 的显示/隐藏

## 修复方案

### 1. 添加状态管理

在 `PermissionList.tsx` 中添加 Modal 显示状态：

```typescript
const [createPermissionModalVisible, setCreatePermissionModalVisible] = useState(false)
const [createRouteModalVisible, setCreateRouteModalVisible] = useState(false)
```

### 2. 为按钮添加 onClick 处理

```typescript
// 创建权限按钮
<Button 
  type="primary" 
  icon={<PlusOutlined />}
  onClick={() => setCreatePermissionModalVisible(true)}
>
  创建权限
</Button>

// 创建路由规则按钮
<Button 
  type="primary" 
  icon={<PlusOutlined />}
  onClick={() => setCreateRouteModalVisible(true)}
>
  创建路由规则
</Button>
```

### 3. 创建 Modal 组件

需要创建两个新的 Modal 组件：

- `CreatePermissionModal.tsx` - 创建权限的表单
- `CreateRouteModal.tsx` - 创建路由规则的表单

参考 `SystemList.tsx` 和 `RoleList.tsx` 的实现模式：

- 使用 Ant Design 的 Modal 和 Form 组件
- 使用 `useMutation` 处理提交
- 成功后刷新列表并关闭 Modal

### 4. 在页面中引入并使用 Modal

```typescript
{/* 创建权限Modal */}
<CreatePermissionModal
  visible={createPermissionModalVisible}
  onClose={() => setCreatePermissionModalVisible(false)}
  onSuccess={() => {
    queryClient.invalidateQueries({ queryKey: ['permissions'] })
  }}
/>

{/* 创建路由规则Modal */}
<CreateRouteModal
  visible={createRouteModalVisible}
  onClose={() => setCreateRouteModalVisible(false)}
  onSuccess={() => {
    queryClient.invalidateQueries({ queryKey: ['routes'] })
  }}
/>
```
